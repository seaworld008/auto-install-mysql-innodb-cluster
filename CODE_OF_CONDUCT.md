# Code of Conduct

本项目希望为使用者、维护者和贡献者提供专业、友善、可持续的协作环境。

## 我们鼓励

- 用事实、日志、复现步骤和上下文讨论问题。
- 尊重不同经验水平的使用者。
- 对风险、限制和不确定性保持清晰表达。
- 在技术分歧中聚焦问题本身，而不是个人。
- 主动帮助改进文档、示例、测试和排错流程。

## 我们不接受

- 人身攻击、羞辱、歧视、骚扰或威胁。
- 泄露他人或组织的敏感信息。
- 在公开 Issue 中粘贴真实密码、Token、私钥或生产环境敏感配置。
- 持续发布与项目无关的广告、垃圾内容或恶意链接。
- 明知有破坏性风险仍诱导他人直接在生产环境执行危险命令。

## 执行方式

维护者可以编辑、隐藏或删除不符合本准则的内容，也可以限制持续破坏协作环境的账号参与项目。

如遇安全漏洞，请按 [SECURITY.md](SECURITY.md) 的方式报告，不建议在公开 Issue 中披露可被利用的细节。
